Claude Code v2.1.83 - March 25, 2026 (76 changes)

Full changelog | Official docs

New Features

  • Transcript search - Press / in transcript mode (Ctrl+O) to search, n/N to jump between matches
  • Credential scrubbing - CLAUDE_CODE_SUBPROCESS_ENV_SCRUB=1 strips API keys from subprocesses (bash, hooks, MCP servers)
  • CwdChanged/FileChanged hooks - Reactive environment management (e.g., direnv auto-loading)
  • sandbox.failIfUnavailable - Exit with error if sandbox can't start instead of running unsandboxed
  • Agent initialPrompt - Agents can auto-submit a first turn via frontmatter
  • Image chips - Pasted images get [Image #N] labels at cursor for positional reference
  • Plugin configuration - Plugins can prompt for config at enable time, sensitive values stored in keychain

Performance

  • --bare -p is ~14% faster to first API call
  • --resume uses less memory on large sessions
  • Plugins load from disk cache instead of re-fetching
  • Non-streaming fallback cap: 21k to 64k tokens, timeout 120s to 300s
  • Startup ~4s faster (voice module + MCP config fetch fixes)

Notable Fixes

  • Mac sleep prevention (caffeinate not terminating) - fixed
  • Subagents invisible after context compaction causing duplicates - fixed
  • rg | wc -l hanging in sandbox on Linux - fixed
  • Memory leak in remote sessions (tool IDs accumulating) - fixed
  • MCP tool calls hanging on SSE connection drop - fixed
  • Scrollback jumping when thinking starts/stops - fixed
  • Tool result files ignoring cleanupPeriodDays - fixed

Other

  • /status works mid-response now
  • Ctrl+L forces full screen redraw
  • WebFetch identifies as Claude-User for robots.txt
  • MEMORY.md index truncates at 25KB/200 lines
  • "Stop all agents" keybinding: Ctrl+F to Ctrl+X Ctrl+K
  • [VSCode] "Not responding" spinner after 60s, new rewind picker
